# 方針

中学生。フロント中心。思いついた時だけ開発する。

## リポジトリ運用
- 思いついた時だけ作る
- 基本は放置
- 継続開発はしない

## 開発スタイル
- フロントエンド中心
- バックエンドは使わない
- コードはAI生成
- 理解・修正は基本しない

## 挙動
- 動いたら終了
- 動かなかったらAIに聞く
- 必要なら詰める

## 自動化
- GitHub Actionsで5分ごとにコミットを実行
- 草（contribution graph）維持目的
- 実際の開発とは無関係な更新も含む
- リポジトリの“生存アピール”として動作

## 技術
- HTML / CSS / JavaScript
- Python（気分）
- GitHub Actions（草維持インフラ）

## 目的
- 思いつきを形にする
- リポジトリは放置前提
- 草を維持する
